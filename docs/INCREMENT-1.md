# Vertical increment 1 — Auth + Pages + Drafts + API + Panel + Astro

First functional cut, **verified end-to-end** (real Postgres + browser). Covers the
skeleton of Milestones 1–3 of the [roadmap](08-roadmap.md).

## What it includes

| Layer | Package/App | Content |
|------|-------------|-----------|
| Contracts | `@astrocms/contracts` | Types + Zod for auth, entries/pages, session, errors |
| Data | `@astrocms/cms-database` | Drizzle schema (10 tables), `0000` migration, idempotent seed |
| Auth | `@astrocms/cms-auth` | argon2id, session tokens (hashed), admin/editor RBAC |
| Domain | `@astrocms/cms-core` | Login/session; entries with **versions**, **transition table** (draft↔published↔archived), publish/unpublish/revisions/restore |
| API | `apps/cms-server` (Fastify) | `/api/v1` auth + pages + public + `/healthz`; HttpOnly cookies; CSRF double-submit; rate limit on login |
| SDK | `@astrocms/cms-sdk` | Typed client (browser + SSR) |
| Public | `apps/astro-demo` (Astro SSR) | Renders a published page by slug; 404 if it doesn't exist |
| Panel | `apps/cms-admin` (React) | Login, list, create/edit page, publish, revisions (TanStack Router/Query + RHF + Zod) |

## Verification performed

- **Unit + integration:** 20 passing tests (pure domain + HTTP integration against real Postgres).
  The integration test covers: successful/failed login, mandatory CSRF, draft **hidden** from the public API,
  publish → visible, edit → new version, revisions, restore.
- **Astro SSR:** published page rendered from the public API; non-existent slug → 404.
- **Panel in browser:** login → session (HttpOnly cookie via proxy) → real list → create page
  (autogenerated slug, draft, v1) → editor with publish/revisions; logout revokes the session.
- **Strict typecheck** clean across all 9 projects. No file exceeds 500 LOC.

## Getting started (local)

```bash
# 1. Requirements: Node ≥20, pnpm ≥9, Docker.
cp .env.example .env            # set SESSION_SECRET
pnpm install

# 2. Infra (Postgres on host port 5433; avoids clashing with a native PG on 5432)
docker compose up -d postgres

# 3. Migration + seed (demo users)
export DATABASE_URL="postgres://astrocms:astrocms@127.0.0.1:5433/astrocms"
pnpm --filter @astrocms/cms-database db:migrate
pnpm --filter @astrocms/cms-database db:seed
#   admin@astrocms.local / Admin!2345   ·   editor@astrocms.local / Editor!2345

# 4. Start (3 terminals, or in the background)
pnpm --filter @astrocms/cms-server start     # API on :3000
pnpm --filter @astrocms/cms-admin  dev       # panel on :5173 (proxy /api → :3000)
CMS_API_URL=http://127.0.0.1:3000/api/v1 pnpm --filter @astrocms/astro-demo dev   # site on :4321
```

- Panel: http://localhost:5173 → sign in as admin → create and publish a page.
- Public site: http://localhost:4321/<slug> shows the published page (SSR).

## Design notes applied (Architecture Advisor)

- **Behavior:** entry lifecycle modeled as a **status enum + transition table**
  (`entry-transitions.ts`); no workflow engine (no timers/human tasks yet).
- **Decision:** authorization guards (RBAC) at the HTTP edge; the `pages.publish` permission
  separates publishing from editing.
- **Execution:** *command handlers* in `cms-core` with **transactions** per state change
  (create version + move pointer) — no outbox yet (no external effects; webhooks come later).
- **Hexagonal / infrastructure-agnostic:** the domain knows nothing about HTTP or the DB provider; `DATABASE_URL`
  points to any Postgres. `pingDb` keeps drizzle from leaking to the edge.
